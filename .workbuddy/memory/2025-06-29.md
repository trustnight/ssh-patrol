## 修复 EXE 运行三大问题（tray_app.py）

### 1. 日志路径修复
- **问题**：`tray_app.py` 第25行硬编码 `APPDATA/.patrol_assistant/logs/`，EXE运行时日志散落各处
- **修复**：改为 `Path(get_data_dir()) / "log" / "patrol.log"`，与 `config.py` 统一
  - EXE模式 → `exe所在目录/log/patrol.log`
  - 开发模式 → `run_dev/log/patrol.log`（由 `PATROL_DEV_OUTPUT_DIR` 控制）

### 2. 自动打开浏览器
- **问题**：EXE模式下 `start_tray()` 没有打开浏览器，只有托盘菜单能手动打开
- **修复**：`start_tray()` 中服务器线程启动后，加 daemon 线程延迟2秒自动打开
- `_fallback_run_server()` 也补了自动打开逻辑（pystray不可用时的回退路径）

### 3. "查看运行日志"打开CMD窗口
- **问题**：`_view_logs()` 用 `os.startfile(log_file)` 以记事本打开，无法实时看日志
- **修复**：改为 `subprocess.Popen` 打开新CMD窗口，先 `type` 历史日志，再用 `powershell Get-Content -Wait -Tail 10` 实时tail

### 项目定位
网络设备批量巡检Web应用 + 截图自动命名工具，面向网络运维工程师。

### 技术栈
- **后端**: FastAPI + SQLite + Paramiko/AsyncSSH (Python 3.7.10, conda env: py3710)
- **前端**: Vue3 + Element Plus + Vite
- **打包**: PyInstaller 单文件exe
- **截图**: PIL (ImageGrab) + pynput 全局快捷键

### 核心架构
- 后端入口 `backend/app/main.py`，通过 `create_app()` 注册路由、WebSocket、CORS、静态文件
- Database 单例模式 (`app/database.py`)，每次操作独立 connect/close
- Service 单例模式：patrol_service、task_manager、screenshot_service
- 路径兼容：`get_exe_dir()` / `get_resource_dir()` 兼容开发/PyInstaller打包
- 密码加密：AES加密存储 (`crypto_utils.py`)

### 数据库 (SQLite: patrol.db)
7张表：devices, commands, manufacturer_commands, patrol_history, screenshot_urls, screenshot_history, screenshot_tasks, screenshot_task_devices

### 前端路由
/ → /patrol (巡检任务列表), /patrol/:taskId (任务详情), /ssh-terminal, /template-manage, /device-manage, /screenshot

### 支持的设备厂商
Hillstone(山石), Huawei(华为), Topsec(天融信) — 含预置的命令模板(普通巡检/深度巡检/自定义)

### 巡检流程
task_manager管理任务状态 → patrol_service通过Paramiko SSH连接设备 → 执行命令模板 → WebSocket实时推送日志 → 结果保存到 log/ 目录

- `database.py` — add_device 返回 `(device_id, 'insert'/'update')` 元组

- `DeviceManage.vue` — 导入统计 "新增N台 + 覆盖更新N台"，手动表单也提示覆盖

- **问题1**: `v-else` 语法陷阱导致连接/断开按钮同时出现（`v-else` 配对的是历史下拉的 `v-if` 不是连接按钮的 `v-if`）

- **修复**: 用 `<template v-if="!connected">` 包裹连接区，断开按钮独立 `v-if="connected"`；form 加 `@keydown.enter.prevent="handleConnect"`
